2. CSV Format

The package reads 2-column CSV files with this format:

time [months], sunspot_number [count]
1, 12.4
2, 15.1
3, 14.8

Rules:

  • Row 1 is always the header: name [unit]
  • Column 1 = independent variable (time, index, position, …)
  • Column 2 = observable (what you measured)
  • Unitless columns: use empty brackets → flux []
  • Space before bracket is optional: time[months] works too
Blank rows in your CSV? Clean them first: 
import pandas as pd
pd.read_csv('raw.csv', skip_blank_lines=True).dropna(how='all').to_csv('clean.csv', index=False)

3. The Analysis Pipeline

Step A

Import the package

import whitenoise as wn
Step B

Load your data

time, values, meta = wn.read_csv('earthquake_ph.csv')
print(meta)   # shows column names, units, number of points
Step C

Inspect the series

wn.plot_series(time, values, metadata=meta)
Step D

Compute and plot the MSD

lags, msd = wn.compute_msd(values)
wn.plot_msd_empirical(lags, msd, metadata=meta)

MSD = Mean Square Displacement. The shape of this curve tells you what kind of memory the system has.

Step E

Choose a model and analyze

result = wn.analyze('earthquake_ph.csv', model='exponential')
result.summary()

This runs the full pipeline and prints the fitted memory parameter μ with confidence intervals and R².

Step F

Visualize results

wn.plot_diagnostics(result)   # 2×2 figure: series, MSD, PDF, parameters
Step G

Export your result

result.to_csv('my_result.csv')
Step H

Compare multiple datasets

comp = wn.compare({
    'Earthquakes': 'earthquake_ph.csv',
    'Sunspots':    'sunspot_monthly.csv',
}, model='exponential')

comp.summary_table()        # DataFrame of μ values side by side
wn.publish_comparison(comp) # Bar chart with 95% CI error bars
Step I

Batch process a folder

batch = wn.batch_analyze('my_data_folder/', model='cosine')
batch.to_csv('batch_results.csv')

4. The Memory Parameter μ

The central result of each analysis is the memory parameter μ, extracted by fitting the MSD curve.

μ rangeRegimeWhat it means
μ < 0.95SubdiffusiveSystem resists spreading — fluctuations slow down over time
0.95 – 1.05Near-BrownianWeak or no memory — behaves like an ordinary random walk
1.05 – 2.0SuperdiffusiveFluctuations reinforce each other — fast spreading
μ > 2.0HyperballisticExtremely persistent dynamics — very strong memory
fBm model: Reports the Hurst exponent H instead of μ. H = 0.5 → Brownian; H < 0.5 → subdiffusive; H > 0.5 → superdiffusive.

5. Available Models

Model nameParametersBest for
'exponential'μ, βDecay-type dynamics, inter-event times (earthquakes)
'cosine'μ, νOscillatory systems (sunspots, CO₂, climate)
'sine'μ, νOscillatory — like cosine, wider valid range
'fbm'HClassical fractional Brownian motion
'dna'a, b, cRestricted diffusion (DNA loop distances)

How to choose:

  • Start with 'exponential' — most robust for most datasets
  • Check R²: ≥ 0.8 good  |  0.5–0.8 moderate  |  < 0.5 try another model
  • Low R²? Try a different model: re-run wn.analyze(..., model='cosine')
  • See all models: wn.list_models()

7. Optional Preprocessing

These steps are optional — start without them and add only if needed.

# Remove a linear trend (to extract fluctuations)
fluct = wn.detrend(values, method='linear')

# Normalize if data has extreme scale (makes N parameter stable)
scaled = wn.normalize(values, method='zscore')

# Smooth noisy data
smooth = wn.smooth(values, window=7)

# Then analyze preprocessed data
result = wn.analyze(fluct, model='cosine', label='Detrended data')
When to normalize? If your fit summary shows N = 0.0000, your data amplitude may be very small. Normalize first: wn.normalize(values, method='zscore'), then pass the result to wn.analyze(). The memory parameter μ is not affected by normalization.

8. Analyzing Your Own Data

Step 1 — Prepare your CSV file

Your file must follow the 2-column format from Section 2:

time [unit], variable_name [unit]
1, 23.5
2, 24.1
...
  • Header row required — format: name [unit]
  • Column 1 = independent variable (time, index, position…)
  • Column 2 = your observable (what you measured)
  • No blank rows in the middle
Blank rows in your file? Clean them first in a separate cell:
import pandas as pd
pd.read_csv('raw.csv', skip_blank_lines=True).dropna(how='all').to_csv('clean.csv', index=False)

Step 2 — Upload to Colab

Run this cell to open the file upload dialog:

Use MY_FILE (the variable from Step 2) wherever the code asks for a file path:

Or just pass the filename as a string directly if you already know the name:
