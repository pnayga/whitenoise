whitenoise — User Guide

Stochastic White Noise Analysis with Memory (SWNA)
Package version 0.2.0 · Python ≥3.9 · March 14, 2026

1. Overview

whitenoise is a research package implementing Stochastic White Noise Analysis (SWNA) with Memory. The framework models a fluctuating observable as a stochastic integral driven by white noise, where a memory kernel encodes how past fluctuations influence future dynamics.

The central quantity is the Mean Square Displacement (MSD), which characterises how the spread of a fluctuating observable grows with lag Δ: 

MSD(Δ) = (1 / (N − Δ)) · ∑i=0N−Δ−1 [x(i+Δ) − x(i)]²

Fitting a theoretical SWNA model to the empirical MSD yields the memory parameter μ, which quantifies the strength and direction of memory in the system. The four diffusion regimes are:

RegimeCriterion (non-fBm models)fBm criterion (H)Physical interpretation
Subdiffusiveμ < 0.95H < 0.475Past fluctuations resist change — slow spreading
Near-Brownian0.95 ≤ μ ≤ 1.050.475 ≤ H ≤ 0.525Weak or no memory — ordinary random walk
Superdiffusive1.05 < μ ≤ 2.0H > 0.525Past reinforces future — fast spreading
Hyperballisticμ > 2.0—Extreme persistence — strongly correlated dynamics
Scope: The package accepts any sequential 1D observable — not just time series. Column 1 is the independent variable (time, index, frequency, distance, occurrence number, etc.); column 2 is the observable. The term “time” is used loosely throughout this guide.

3. Quickstart

import whitenoise as wn

# --- Analyze a single CSV ---
result = wn.analyze('sunspot.csv', model='exponential')
result.summary()           # prints formatted parameter table
result.plot_all()          # three-panel figure: series, MSD, PDF

# --- Inspect data before fitting ---
time, values, meta = wn.read_csv('sunspot.csv')
wn.plot_series(time, values, metadata=meta)      # raw series
lags, msd = wn.compute_msd(values)
wn.plot_msd_empirical(lags, msd, metadata=meta)  # MSD shape check

# --- Optional detrending before analysis ---
fluct = wn.detrend(values, method='moving_average', window=11)
result = wn.analyze(fluct, model='cosine', label='Sunspot fluctuations')

# --- Compare multiple datasets ---
comparison = wn.compare({
    'Sunspots':    'sunspot.csv',
    'Earthquakes': 'earthquake_ph.csv',
}, model='exponential')
print(comparison.summary_df)
wn.publish_comparison(comparison)

# --- Batch process a folder ---
batch = wn.batch_analyze('data/xray_binaries/', model='cosine')
batch.to_csv('results.csv')

# --- Try all models on one file ---
search = wn.batch_model_search('sunspot.csv')
print(search.summary_df[['model', 'r_squared']].to_string())

Note on normalization: Pre-analysis normalization is not standard in the published SWNA work. The normalization scalar N in the fit (N · msd_model) absorbs data amplitude and scale automatically — the memory parameter μ is unaffected by amplitude. Normalization is only useful in rare edge cases of extreme numerical scale differences.

7. Preprocessing (Optional)

These helpers are optional. Call them manually before analyze() if your data has a non-stationary trend or needs smoothing. The pipeline never calls them automatically unless you pass detrend_method= to analyze().

When to detrend: The SWNA framework assumes the observable represents fluctuations around some baseline. If your data has a clear long-term trend (e.g. a rising CO&sub2; baseline, a linear drift), remove it first. If the data is already fluctuating around zero, no detrending is needed.

# Sunspot detrending — remove slow baseline cycle
time, values, meta = wn.read_csv('sunspot.csv')

# CO2 polynomial detrend
time, co2, meta = wn.read_csv('co2.csv')
fluct = wn.detrend(co2, method='polynomial', poly_order=2)

There are also two advanced detrending functions not exposed in the standard pipeline but available in utils/preprocess.py:

  • detrend_psd(values, threshold=0.5) — Removes dominant periodic components via Power Spectral Density analysis (PSD filtering). Useful for data with strong deterministic periodic cycles.
  • detrend_fingerprint(values, cycle_length=None, trough_indices=None) — Cycle averaging (“fingerprinting”) detrending: aligns repeated cycles at their maxima, averages them, then subtracts. Useful when cycles have similar shapes but vary in timing.

8. Core Functions

compute_msd()

compute_msd(x, max_lag=None, normalize=False) -> tuple[np.ndarray, np.ndarray]

Compute the empirical Mean Square Displacement of a 1D time series.

Formula: MSD(Δ) = (1 / (N−Δ)) · ∑i [x(i+Δ) − x(i)]²

ParameterTypeDefaultDescription
xarray-like (1D)requiredFluctuating observable. Accepts ndarray, pd.Series, list. Isolated NaN values are replaced by linear interpolation before computation.
max_lagint or NoneNoneMaximum lag. Defaults to N//2. Hard-capped at N//2 (beyond N/2, each MSD estimate uses fewer than N/2 data pairs and becomes unreliable).
normalizeboolFalseIf True, divide all MSD values by MSD[0] (lag=1). Preserves the scaling exponent μ while setting the initial value to 1.

Returns: (lags, msd) — both shape (max_lag,). lags are integer steps 1…max_lag.

Raises: ValueError if input is not 1D (message: ✗ Input must be 1D. Got shape …); if fewer than 10 points (message: ✗ Need at least 10 data points. Got …); if more than 50% NaN (message: ✗ Too many missing values (…% NaN). Check your data.).

time, values, meta = wn.read_csv('sunspot.csv')
lags, msd = wn.compute_msd(values)
print(lags.shape)   # (N//2,) e.g. (150,) for 300 points

# Inspect shape before fitting
wn.plot_msd_empirical(lags, msd, metadata=meta)

result = wn.fit_msd(lags, msd, model='exponential')

    print(result.params['mu'])    # e.g. 1.15

9. Models

The MODELS registry contains 19 models (18 theoretical MSD formulas from the SWNA reference literature, plus the DNA plateau model from Violanda et al. 2019). Model selection is always manual — the researcher specifies model=. The package never auto-selects. Run wn.list_models() to see all models and their current status.

Model selection guidance

One model per run: analyze() fits exactly one model. Selecting the right model is the researcher’s responsibility. If unsure, start with model='exponential' (most commonly used in published SWNA studies). If your data shows clear oscillatory structure, try model='cosine'. Use wn.batch_model_search('mydata.csv') to run all available models and compare R² values side by side.
R² rangeInterpretationAction
≥ 0.8Good fitReport μ with 95% CI. Results are reliable.
0.5 – 0.8Moderate fitCaution. Report but note uncertainty. Consider another model.
< 0.5Poor fitModel is likely wrong. The package prints a ⚠ warning and suggests alternatives. Try batch_model_search().

10. Plotting

The package provides two sets of plotting functions. Exploratory plots (viz/explore.py) are fast and annotated — designed for interactive inspection. Publication plots (viz/publish.py) apply strict rcParams styling (serif font, inward ticks, dpi=150) suitable for papers and reports.

Headless server note: import whitenoise as wn does not import matplotlib at package level. Viz functions import matplotlib lazily on first call. On headless servers (no display), set the backend before calling any viz function: import matplotlib; matplotlib.use("Agg")

Exploratory plots — viz/explore.py

All exploratory functions return a matplotlib.figure.Figure. All accept ax=None (create new figure) and show=True (call plt.show()).

13. Advanced Usage

Running the pipeline step by step

For full control, call each step individually:

import whitenoise as wn
import numpy as np

# Step 1: Load
time, values, meta = wn.read_csv('sunspot.csv')

# Step 2: Inspect
fig = wn.plot_series(time, values, metadata=meta, show=False)

# Step 3: Detrend (manual decision)
fluct = wn.detrend(values, method='moving_average', window=11)

# Step 4: Compute MSD
lags, msd = wn.compute_msd(fluct)

# Step 5: Inspect MSD shape
wn.plot_msd_empirical(lags, msd, metadata=meta)

# Step 6: Fit with custom initial guess
fit = wn.fit_msd(lags, msd, model='exponential',
                 p0=[1.5, 0.05],
                 max_lag_fraction=0.8)

if fit is not None:
    print(fit.summary())
    print(f'mu = {fit.params[\"mu\"]:.4f}')
    lo, hi = fit.confidence_intervals['mu']
    print(f'95% CI: ({lo:.4f}, {hi:.4f})')

Custom p0 and bounds

If the default initial guesses fail (fit returns None or R² is low), supply p0 and/or bounds based on domain knowledge. Remember: N is always appended internally — do not include it in p0 or bounds. 

# Custom p0 for cosine model (physical params mu, nu only; N appended internally)
result = wn.analyze('coral.csv', model='cosine',
                    fit_kwargs={'p0': [4.5, 0.003]})

# Custom p0 and bounds
result = wn.analyze('coral.csv', model='cosine',
                    fit_kwargs={
                        'p0':    [4.5, 0.003],
                        'bounds': ([0.5, 1e-9], [10.0, 1.0]),
                    })

Methodological note: fitting method

Current fitting uses unweighted SSR (ordinary least squares), consistent with the published SWNA prior published results. A more statistically rigorous alternative would be Weighted Least Squares (WLS) with weights wΔ = N − Δ (large-lag MSD estimates are noisier, computed from fewer data pairs). WLS is not yet implemented but references for the approach include: Michalet (2010) and Vestergaard et al. (2014).

15. Genomics Extension (wn.genomics)

The wn.genomics submodule bridges standard genomic sequence data (FASTA format from NCBI, BOLD, or any public database) into the whitenoise SWNA pipeline. It implements the nucleotide separation distance approach described in Violanda et al. (2019), extended to include all 16 pairwise transition distances.

No additional dependencies required. The genomics submodule uses only NumPy and the Python standard library. BioPython is optional — a convenience adapter is provided for users who already have it.

The stochastic variable

Following Violanda et al. (2019), the stochastic variable is the separation distance between nucleotides along the genome. For a given pair (from_nuc → to_nuc):

  • Same-type (A→A, G→G, etc.): distance between consecutive occurrences of the same nucleotide. Reproduces the paper's method exactly.
  • Cross-type (A→C, A→G, etc.): from each occurrence of from_nuc, the distance to the very next occurrence of to_nuc downstream (nearest-next rule). This is a generalization not in the original paper.

Across all four nucleotides, this produces a 4×4 matrix of 16 distance sequences. Each sequence is the stochastic observable fed into wn.compute_msd() and wn.analyze().

Supported input formats

The submodule accepts any standard FASTA file, including:

  • COI barcodes (~658 bp) from NCBI GenBank or BOLD
  • Whole mitochondrial genomes (~15–20 kb)
  • Complete bacterial/organism genomes (Mbp–Gbp)
  • Multi-record FASTA files (one record returned per sequence)

read_fasta()

records = wn.genomics.read_fasta(path)

# Returns a list of dicts, one per FASTA record:
# {
#   'id':          str   accession/identifier (first token after '>')
#   'description': str   full header line
#   'sequence':    str   uppercase DNA string
#   'length':      int   total sequence length
#   'n_ambiguous': int   count of non-ACGT characters (W, N, R, etc.)
# }
Ambiguous bases (W, N, R, Y, etc.) are kept in the sequence and a warning is printed. They are silently skipped during position lookup in the distance functions, so they do not cause errors. Use wn.genomics.filter_sequence(seq) to remove them first if you prefer.

get_transition_distances()

distances = wn.genomics.get_transition_distances(sequence, from_nuc, to_nuc)
# Returns: np.ndarray of int64 distances

Extracts the distance array for a single (from_nuc, to_nuc) pair. Pass the result directly to wn.compute_msd() or wn.analyze().

Performance: fully vectorized using np.frombuffer + np.searchsorted. Complexity is O(N log M) — handles whole bacterial genomes (>2 Mbp) in milliseconds per pair.

get_all_transition_distances()

matrix = wn.genomics.get_all_transition_distances(sequence)
# Returns: dict with (from_nuc, to_nuc) tuple keys
# e.g. matrix[('A', 'C')] -> np.ndarray of distances

wn.genomics.summarize_matrix(matrix)   # prints count table

Computes all 16 pairs in one call, scanning the sequence only 4 times (once per nucleotide). More efficient than calling get_transition_distances() 16 times individually.

Complete genomics workflow

import whitenoise as wn

# 1. Load FASTA from NCBI or any source
records = wn.genomics.read_fasta('Synechococcus_elongatus.fasta')
seq = records[0]['sequence']
print(f"Loaded {records[0]['id']}: {records[0]['length']} bp")

# 2. Optional: remove ambiguous bases
seq = wn.genomics.filter_sequence(seq)

# 3. Check all 16 transition pairs
matrix = wn.genomics.get_all_transition_distances(seq)
wn.genomics.summarize_matrix(matrix)

# 4. Analyze one pair with SWNA
distances = matrix[('A', 'A')]   # same-type: reproduces Violanda et al.
result = wn.analyze(distances, model='dna', label='A->A (S. elongatus)')
result.summary()

# 5. Compare all 16 pairs
all_results = {}
for (from_n, to_n), dist in matrix.items():
    if len(dist) >= 50:   # skip pairs with too few observations
        label = f'{from_n}->{to_n}'
        all_results[label] = dist

comparison = wn.compare(all_results, model='dna')
comparison.summary_table()
wn.publish_comparison(comparison)

17. Known Limitations

1. Stub models. Nine models in the MODELS registry raise NotImplementedError when called: exp_whittaker, bessel_K, hypergeom_F1, bessel_I, hypergeom_3F2, csc_power, cot_power, bessel_pair, bessel_pair2. Use wn.list_models() to see current status.
2. Unweighted fitting. The current fitting uses ordinary least squares (unweighted SSR). Statistically, WLS with weights wΔ = N − Δ would be more rigorous (large-lag MSD estimates are noisier). Not implemented, but kept consistent with prior published results from the Bernido group.
3. Parallel n_jobs > 1 requires CSV paths. When n_jobs > 1 is specified in batch_analyze(), all jobs must use CSV file paths. In-memory arrays cannot be reliably passed to parallel worker threads.
4. Oscillatory model NaN regions. The cosine and sine models return nan where cos(νT/2) ≤ 0 or sin(νT/2) ≤ 0 respectively (the formula produces a non-positive MSD there, which has no physical meaning). If your MSD plot shows gaps, this is expected. Consider using a smaller lag range with max_lag_fraction, or try a different model.
5. Minimum data length. compute_msd() requires at least 10 data points. Fewer than ~50 points will produce very noisy MSD estimates and unreliable fits. Published results typically use N ≥ 200 points.
