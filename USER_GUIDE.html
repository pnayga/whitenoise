whitenoise — User Guide

Stochastic White Noise Analysis (SWNA) — Hida–Bernido Framework
Package version 0.1.0 · Python ≥3.9

1. Overview

whitenoise is a research package for analysing the memory and diffusion properties of sequential observational data using the Stochastic White Noise Analysis (SWNA) with Memory framework developed by Bernido & Carpio-Bernido.

The central object of SWNA is the Mean Square Displacement (MSD), which characterises how the spread of a fluctuating observable grows with lag time Δ: 

MSD(Δ) = (1 / (N−Δ)) · ∑[x(i+Δ) − x(i)]²    (summed over i = 0 … N−Δ−1)

Fitting a theoretical model to the empirical MSD yields the memory parameter μ, which quantifies how strongly the system’s past fluctuations influence its future. The table below summarises the four diffusion regimes:

RegimeCriterionPhysical interpretation
Subdiffusiveμ < 0.95Past fluctuations resist future change — slow spreading
Near-Brownian0.95 ≤ μ ≤ 1.05Weak or no memory — ordinary random walk
Superdiffusive1.05 < μ ≤ 2.0Past fluctuations reinforce future ones — fast spreading
Hyperballisticμ > 2.0Extreme persistence — strongly correlated dynamics
Scope: The package accepts any sequential 1D observable, not just time series. Column 1 is the independent variable (time, index, lag, distance …); column 2 is the observable. The word “time series” is used loosely throughout.

3. CSV Input Format

The package reads 2-column CSV files with a required header row. The header encodes column names and physical units in the format name [unit]: 

time [months], sunspot_number [count]
1, 12.4
2, 15.1
3, 14.8

Column 1Independent variable (time, index, frequency, distance, …). Always.
Column 2Observable to analyse. Always.
Header formatname [unit] — space before bracket is optional: co2[ppm] also works.
Unitless columnUse empty brackets: flux [] or value [unitless]
No bracketsAlso accepted: time, distance — unit defaults to empty string.
Multi-column CSVColumn 1 = independent variable; all other columns = separate observables (batch mode).

time, values, meta = wn.read_csv('sunspot.csv')

results = wn.read_csv_multi('climate.csv')

4. The Full Pipeline

CSV file → read_csv() → detrend() / normalize() → analyze() → AnalysisResult → .summary() / plot / export

4.1 Quickstart — CSV path

result = wn.analyze('sunspot.csv', model='exponential')
result.summary()                # prints formatted parameter table
wn.plot_diagnostics(result)     # 2×2 overview: time series, MSD, PDF, parameters

4.2 Quickstart — NumPy array

import numpy as np
data = np.loadtxt('raw_data.txt')
result = wn.analyze(data, model='cosine', label='My System')
result.summary()
When passing an array, label= is required — it becomes the dataset name in all outputs and plots. When passing a CSV path, the dataset name is taken from the filename (e.g. 'sunspot.csv' → 'sunspot').

5. Optional Preprocessing

Preprocessing functions are never called automatically by analyze(). The researcher applies them manually before analysis when scientifically appropriate.

When to detrend: Only if the data has a deterministic trend that is not part of the stochastic dynamics you want to study (e.g. a linear warming trend in temperature data). Detrending changes the data; use detrend_method=None (the default) when in doubt.

5.1 detrend()

MethodDescription
'linear'Subtract best-fit straight line (degree-1 polynomial).
'polynomial'Subtract degree-n polynomial fit. Set poly_order=n.
'mean'Subtract global mean only.
'moving_average'Subtract a moving-average trend. Set window=k.
'psd'Bernido detrending §3.3 — power spectral density subtraction.
'fingerprint'Bernido detrending §3.4 — stochastic fingerprint method.
fluct = wn.detrend(values, method='linear')
fluct = wn.detrend(values, method='polynomial', poly_order=2)
fluct = wn.detrend(values, method='moving_average', window=11)

5.2 normalize()

MethodOutput rangeUse case

'minmax'[0, 1]Bounded positive data.
'mean'mean=1Preserve scale, center around 1.
'maxabs'[−1, 1]Oscillating data; preserves zero and sign.

5.3 smooth()

smoothed = wn.smooth(values, window=7, method='moving_average')
smoothed = wn.smooth(values, window=7, method='gaussian')

5.4 Full preprocessing workflow example

time, values, meta = wn.read_csv('co2_keeling.csv')

# Inspect raw data first
wn.plot_series(time, values, metadata=meta)

# Remove quadratic trend (CO2 has a nonlinear growth trend)
fluct = wn.detrend(values, method='polynomial', poly_order=2)
wn.plot_series(time, fluct, metadata=meta, title='CO2 fluctuations')

# Analyse the residuals
result = wn.analyze(fluct, model='cosine', label='CO2 fluctuations')
result.summary()

7. FitResult — Fit Output

result.fit is a FitResult dataclass, or None if fitting failed. All fits use Levenberg–Marquardt via scipy.optimize.curve_fit. 95% confidence intervals are always included.

AttributeTypeDescription
.paramsdictPoint estimates, e.g. {'mu': 1.15, 'beta': 0.08, 'N': 2.3}
.std_errorsdictStandard errors (sqrt of pcov diagonal)
.confidence_intervalsdict95% CI as (lo, hi) tuples; CI = param ± 1.96 × SE
.r_squaredfloatR² = 1 − SS_res / SS_tot. Closer to 1 is better.
.lags_usedndarrayLags actually used in fitting (after max_lag_fraction truncation)
.msd_fittedndarrayN · msd_model(T) evaluated at lags_used
.modelstrModel name
result.fit.params           # {'mu': 1.15, 'beta': 0.08, 'N': 2.3}
result.fit.r_squared        # 0.9823
result.fit.confidence_intervals['mu']   # (1.11, 1.19)
print(result.fit.summary()) # formatted text block
R² interpretation: ≥0.8 = good fit; 0.5–0.8 = moderate; <0.5 = poor (the package will suggest alternative models automatically).
Fitting method — OLS, not WLS: The fitting uses ordinary least squares (OLS) via scipy.optimize.curve_fit, which minimises the unweighted sum of squared residuals (SSR). This treats all lag points equally, but MSD estimates at large lags are computed from fewer data pairs and are therefore noisier and less reliable than short-lag estimates. A weighted least squares (WLS) approach — down-weighting large lags in proportion to their variance — is not currently implemented.

Practical consequence: If the large-lag end of the empirical MSD is erratic, it can pull the fitted curve away from the well-sampled short-lag region, inflating or deflating R² in a misleading way. Use max_lag_fraction=0.5 to restrict fitting to the first half of lags (the most statistically reliable region) when this occurs.

9. Comparing Multiple Datasets

9.1 compare() — same model, multiple datasets

# Dict input — custom labels, mix of CSV paths and arrays
cr = wn.compare({
    'Earthquakes': 'earthquake_ph.csv',
    'Sunspots':    'sunspot.csv',
    'CO2 Keeling': co2_array,
}, model='exponential')

# List input — labels auto-set from filenames
cr = wn.compare(['chile.csv', 'japan.csv', 'ph.csv'], model='cosine')

# Per-dataset models
cr = wn.compare({'A': 'a.csv', 'B': 'b.csv'},
                model={'A': 'cosine', 'B': 'exponential'})

cr is a ComparisonResult:

cr.summary_df           # pandas DataFrame with one row per dataset
cr.results              # list of AnalysisResult objects
wn.print_comparison(cr) # print ASCII table to terminal
wn.publish_comparison(cr, save_path='comparison.pdf')

9.2 batch_analyze() — three input modes

Modesource argumentExample
A — FolderFolder path stringwn.batch_analyze('data/xray_binaries/')
B — ListList of CSV pathswn.batch_analyze(['a.csv', 'b.csv'])
C — Multi-column CSVCSV path with multiple observableswn.batch_analyze('climate.csv')
cr = wn.batch_analyze('data/xray_binaries/', model='cosine', n_jobs=4)
cr.summary_df       # one row per dataset
wn.export_summary(cr, 'results/batch_summary.csv')
Parallel mode: n_jobs > 1 uses ThreadPoolExecutor. Works for CSV-path inputs only. For in-memory arrays, use n_jobs=1.

9.3 batch_model_search() — find the best model

cr = wn.batch_model_search(
    'sunspot.csv',
    models=['cosine', 'exponential', 'sine', 'fbm'],  # None = all available
)
# Prints R² for each model and reports the best one.
cr.summary_df.sort_values('r_squared', ascending=False)

11. Advanced Usage

11.1 Direct MSD computation

lags, msd = wn.compute_msd(values)
lags, msd = wn.compute_msd(values, max_lag=100)        # cap at 100 lags
lags, msd = wn.compute_msd(values, normalize=True)     # divide by msd[0]
The maximum lag is always capped at N/2 (where N is the number of data points). Beyond N/2, each MSD estimate uses fewer than N/2 pairs and becomes statistically unreliable.

11.2 Direct fitting

fit = wn.fit_msd(lags, msd_empirical, model='cosine')
fit = wn.fit_msd(lags, msd_empirical, model='cosine',
                 p0=[1.2, 0.01],          # initial guess [mu, nu]
                 bounds=([0.01, 1e-9], [5.0, 10.0]),
                 max_lag_fraction=0.5)    # fit only first half of lags

11.3 Model registry

wn.list_models()             # print all 19 models and their status
info = wn.get_model('cosine')
print(info['params'])        # ['mu', 'nu']
print(info['status'])        # 'available'
msd_val = info['msd'](10.0, 1.2, 0.05)   # call MSD function directly

11.4 Headless / server environments

import matplotlib
matplotlib.use('Agg')   # set BEFORE importing whitenoise
import whitenoise as wn

# Now all plot functions work without a display; use save_path= to write files
wn.publish_msd(result, save_path='msd.png', show=False)

14. Known Limitations

  • 9 stub models: exp_whittaker, bessel_K, hypergeom_F1, bessel_I, hypergeom_3F2, csc_power, cot_power, bessel_pair, bessel_pair2 raise NotImplementedError.
  • Parallel batch with arrays: n_jobs > 1 only works with CSV file paths. Pass n_jobs=1 when sources are in-memory arrays.
  • Headless display: On servers without a display, set matplotlib.use('Agg') before importing whitenoise, then use show=False and save_path= arguments.
  • MSD reliability: Only lags up to N/2 are computed. Large-lag estimates (near N/2) have fewer averaging pairs and are noisier — consider using max_lag_fraction=0.5 for fitting if the far end of the MSD is erratic.
  • Unweighted fitting (OLS, not WLS): The fitter minimises the unweighted sum of squared residuals. All lags receive equal weight regardless of how many data pairs contributed to each MSD estimate. Noisy large-lag points can therefore bias the fit and distort R². Weighted least squares (WLS) is not currently implemented — use max_lag_fraction to exclude unreliable large-lag values as a workaround.
