whitenoise — Genomics Guide

SWNA on DNA Sequences · wn.genomics submodule
Framework: Violanda et al. (2019) · Physica Scripta 94, 125006

Overview

This guide covers applying Stochastic White Noise Analysis (SWNA) to genomic DNA sequences. The stochastic variable is the separation distance between consecutive nucleotide occurrences along the sequence. This distance sequence feeds directly into the standard SWNA pipeline via the DNA model.

FASTA → filter_sequence() → extract_pair() save CSV → compute_pair_msd() save CSV → fit_pair() → plot_pair()

Each checkpoint CSV can be reloaded to skip earlier steps. Use refit_pair() to retry fitting from a saved MSD CSV without recomputing anything.

3 · Filtering Ambiguous Bases

wn.genomics.filter_sequence(seq, keep='ACGT') → str 
seq = wn.genomics.filter_sequence(seq_raw)   # removes W, S, N, R, Y, etc.
For most whole mitochondrial genomes and chromosomes, ambiguous bases are < 1% and their removal has negligible effect. Always filter before calling any distance function.

4.2 · Single pair

wn.genomics.get_transition_distances(seq, from_nuc, to_nuc) → np.ndarray 
dist = wn.genomics.get_transition_distances(seq, 'A', 'A')
Pair typeDefinitionScientific basis
Diagonal (A→A, C→C, G→G, T→T) Distances between consecutive same-nucleotide occurrences Direct from Violanda et al. (2019)
Off-diagonal (A→C, A→G, …) From each A, distance to the next C downstream Nearest-next extension
Important: The DNA model was derived for same-type (diagonal) distances. Off-diagonal pairs often yield low R² — this is expected, not a bug. For off-diagonal pairs, try model='exponential' or model='cosine'.

5 · Step-by-Step Single-Pair Pipeline

Use this when you want full control over each step, or when re-fitting a specific pair.

Step 1 — Extract distances

wn.genomics.extract_pair(sequence, from_nuc, to_nuc, save_path=None) → np.ndarray 
dist = wn.genomics.extract_pair(seq, 'A', 'A', save_path='AA_distances.csv')

# Reload later:
_, dist, _ = wn.read_csv('AA_distances.csv')
Large genomes: For a 179 Mbp chromosome, each distance array is ~44 million values (~350 MB as CSV). For such cases, skip saving distances and save only the MSD in Step 2 instead.

Step 2 — Compute MSD

wn.genomics.compute_pair_msd(source, max_lag=1500, save_path=None) → (lags, msd) 
# From a distances array
lags, msd = wn.genomics.compute_pair_msd(dist, max_lag=1500, save_path='AA_msd.csv')

# Or load from saved distances CSV
lags, msd = wn.genomics.compute_pair_msd('AA_distances.csv', max_lag=1500)
Choosing max_lag: The DNA model saturates at approximately L ≈ 3/b. With the published bacterial value b ≈ 0.0024, saturation occurs near L = 1250. max_lag=1500 is a safe default that captures the full curve with a visible flat tail. For organisms with very different b, check the plateau visually first.

Step 3 — Fit a model

wn.genomics.fit_pair(source, model='dna', p0=None, bounds=None, ...) → FitResult 
# From arrays
fit = wn.genomics.fit_pair((lags, msd), model='dna')

# From a saved MSD CSV
fit = wn.genomics.fit_pair('AA_msd.csv', model='dna')

# With custom initial parameters
fit = wn.genomics.fit_pair((lags, msd), model='dna', p0=[27.0, 0.025, 2.0])

Step 4 — Plot

wn.genomics.plot_pair(lags, msd, fit=None, title='', save_path=None) → Figure 
fig = wn.genomics.plot_pair(lags, msd, fit, title='A→A | DNA model', save_path='AA_fit.png')
fig.show()

7 · Re-fitting from a Saved MSD CSV

wn.genomics.refit_pair(msd_csv_path, model='dna', p0=None, bounds=None, max_lag_fraction=1.0, save_path=None, verbose=True) → (FitResult, Figure)

This is the primary tool for improving a poor fit without recomputing distances or MSD.

# Retry with a better initial guess
fit, fig = wn.genomics.refit_pair(
    'results/A_A_msd.csv',
    model='dna',
    p0=[27.0, 0.025, 2.0],
    save_path='results/A_A_refit.png',
)
print(fit.r_squared)

# Try a different model entirely
fit, fig = wn.genomics.refit_pair('results/A_C_msd.csv', model='exponential')

# Fit only first 70% of lags (exclude noisy plateau)
fit, fig = wn.genomics.refit_pair('results/A_A_msd.csv', max_lag_fraction=0.7)

8 · The DNA Model

MSD(L) = N · (a − c · exp(−b · L))
ParameterRoleTypical range (bacterial)
aPlateau height — MSD approaches a as L → ∞3–8
bExponential decay rate — how fast MSD rises to plateau0.001–0.01
cAmplitude of exponential term; must satisfy c < a2–6
NNormalization scalar — fitted automatically~1
No μ parameter. The DNA model uses a, b, c — not the memory parameter μ used in cosine/exponential/sine models. It always describes restricted diffusion (bounded MSD plateau).

Published reference values (Violanda et al. 2019, bacterial genomes):
a ≈ 5.21, b ≈ 0.0024, c ≈ 3.81

9 · Fixing Poor Fits

Symptom: curve looks right but R² is low

The optimizer settled in a local minimum far from the visual optimum. Provide a better initial guess.

How to estimate p0 from the plot:

  1. Read a ≈ the plateau value (where the curve flattens)
  2. Read c ≈ a − MSD(L=1)
  3. Estimate b ≈ 6 / L_plateau where L_plateau is the lag where the curve visually flattens
fit, fig = wn.genomics.refit_pair(
    'A_A_msd.csv',
    model='dna',
    p0=[27.0, 0.025, 2.0],   # your a, b, c estimates
)

Symptom: R² < 0.3 (off-diagonal pairs)

Cross-type distances don't follow the DNA saturating-exponential shape. Try other models:

fit, fig = wn.genomics.refit_pair('A_C_msd.csv', model='exponential')
fit, fig = wn.genomics.refit_pair('A_C_msd.csv', model='cosine')
wn.list_models()   # see all available models

Symptom: plateau not visible at max_lag=1500

The decay rate b is smaller than the bacterial reference. Increase max_lag:

lags, msd = wn.genomics.compute_pair_msd(dist, max_lag=5000, save_path='A_A_msd_5000.csv')

11 · Complete Workflow Example

import whitenoise as wn
import os

# 1. Load and filter
record = wn.genomics.read_fasta_single('chromosome1.fasta')
seq = wn.genomics.filter_sequence(record['sequence'])
print(f"Sequence: {len(seq):,} bp")

# 2. Analyze diagonal pairs step by step
os.makedirs('results', exist_ok=True)

for nuc in ['A', 'C', 'G', 'T']:
    dist = wn.genomics.extract_pair(seq, nuc, nuc)
    lags, msd = wn.genomics.compute_pair_msd(
        dist, max_lag=1500, save_path=f'results/{nuc}_{nuc}_msd.csv'
    )
    fit = wn.genomics.fit_pair((lags, msd), model='dna')
    wn.genomics.plot_pair(
        lags, msd, fit,
        title=f'{nuc}→{nuc} | DNA model',
        save_path=f'results/{nuc}_{nuc}_fit.png'
    )

# 3. Re-fit a specific pair with better p0
fit, fig = wn.genomics.refit_pair(
    'results/A_A_msd.csv',
    model='dna',
    p0=[28.0, 0.025, 3.0],
    save_path='results/A_A_refit.png',
)
print(fit.summary())

12 · Batch Analysis (All 16 Pairs)

For a full 4×4 matrix run, use chromosome1_swna.py. The script:

  1. Reads the FASTA and filters the sequence
  2. Computes all 16 distance arrays (get_all_transition_distances)
  3. Runs compute_msd + fit_msd for each pair (max_lag=1500)
  4. Saves per-pair MSD CSVs (chromosome1_msd_X_Y.csv) for later re-fitting
  5. Writes chromosome1_results.csv and chromosome1_matrix.png

To re-fit any single pair from the batch output:

fit, fig = wn.genomics.refit_pair(
    'chromosome1_msd_A_A.csv',
    model='dna',
    p0=[28.0, 0.025, 3.0],
    save_path='chromosome1_A_A_refit.png',
)
